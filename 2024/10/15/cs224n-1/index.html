第一章 Word2Vec 模型

1. 如何进行 min 操作？

在 Word2Vec 中，min 操作指的是通过梯度下降法（Gradient Descent）对词向量矩阵 UUU 和 VVV 进行优化。我们希望通过最小化损失函数（即负对数似然函数）来找到最优的词向量表示。具体步骤如下：

  • 初始化词向量矩阵 ( U ) 和 ( V ) 为随机值。
  • 对每对中心词 ( c ) 和上下文词 ( o )，计算预测的概率 ( p(o|c) )。
  • 计算损失函数的梯度，即相对于 ( U ) 和 ( V ) 的偏导数。
  • 使用梯度下降法更新 ( U ) 和 ( V ) 的值，直到损失函数达到最小或收敛。

通过这一过程，我们对每个词学习出它的词向量，使得在给定中心词的情况下，能够最大化预测正确的上下文词的概率。

2. 如何获取随机变量 O 和 C？

在 Word2Vec 中，( O ) 和 ( C ) 是语料库中词对的随机变量：

  • ( O ) 表示上下文词（Observed Word），即中心词附近的词。
  • ( C ) 表示中心词（Context Word），即我们想要根据上下文来预测的词。

在实际中，模型通过滑动窗口从文本语料库中提取词对。滑动窗口会在每个句子中移动，取出每个词及其周围的词。例如，在 Skip-gram 模型中，给定中心词 ( c )，模型会预测它左右若干个词作为上下文词 ( o )。

3. 为什么使用概率的负对数？

使用负对数是为了简化优化过程并且符合最大似然估计（Maximum Likelihood Estimation, MLE）的目标：

  • 最大化概率：在模型中，我们希望最大化给定中心词 ( c ) 的情况下预测上下文词 ( o ) 的条件概率 ( p(o|c) )。
  • 负对数的作用：对概率取负对数是常见的做法，因为对数函数是单调递增的，最大化对数概率等价于最大化原始概率。而取负值后，我们的目标变成了最小化损失函数，这与优化算法（如梯度下降）对最小化问题的处理相一致。
  • 简化数学处理：对数函数还能将连乘形式的联合概率转化为加法，使得数学计算更为简单，同时负对数可以避免小数计算带来的数值稳定性问题。

4. 为什么比共现计数方法好？

共现计数方法（如传统的词袋模型或共现矩阵）主要基于词之间在语料库中共同出现的频率，存在以下缺陷：

  • 稀疏性问题：共现矩阵通常是非常稀疏的，因为大多数词对在语料库中很少同时出现，这导致共现计数方法很难捕捉到词之间的潜在关系。
  • 维度灾难：共现计数方法生成的矩阵维度非常高（词汇表大小的平方），处理和存储都非常困难。
  • 不能捕捉语义相似性：简单的共现计数无法充分表达词的语义信息，仅能反映表面上的共现关系。

相比之下，Word2Vec 将词映射到低维的词向量空间，能够通过词向量的几何关系捕捉到词语之间的语义相似性。而且，训练过程中，模型能够通过上下文预测优化词向量，使其能够更好地表示词之间的潜在关联。

5. 为什么并非所有的 ( p(o|c) ) 分布都能被这个模型表示？

Word2Vec 的核心是通过词向量的内积（或点积）来计算词与词之间的相似性，并使用 softmax 函数将其转化为概率。因此，它只能表示基于线性关系的条件概率分布。然而，语言中的词语关系是复杂的，很多情况并不能简单地通过词向量的点积来表示。

  • 局限性：词向量模型假设了简单的线性结构，而现实语言中的词语关系往往具有复杂的非线性特征，这限制了模型对所有可能的 ( p(o|c) ) 分布的表达能力。
  • 这是否是好事或坏事？：从一个角度看，这种限制使得模型更简洁，避免了过拟合问题，并且在计算效率上更优。但从另一个角度看，它也限制了模型在捕捉复杂语言现象时的表现。因此，虽然这种限制是合理的，但在某些情况下会导致模型对复杂语言结构的表达能力不足。

6. 这是好的吗？坏的吗？令人惊讶吗？显而易见吗？

这种限制既可以视为好的，也可以视为不理想的：

  • 好的方面：Word2Vec 的简单结构能够有效处理大规模语料库，并且通过低维表示简化了计算复杂度。它的局限性反而帮助模型避免了对复杂分布的过拟合问题。
  • 不理想的方面：对于某些需要复杂上下文关系的任务，Word2Vec 可能无法捕捉到足够的细节。比如语法关系、长距离依赖等，可能需要更复杂的模型来表达。
  • 是否令人惊讶？：不令人惊讶。因为大多数自然语言处理模型都面临类似的权衡——在模型的复杂度和计算效率之间找到平衡。因此，Word2Vec 这种设计体现了在可计算性与表达能力之间的合理折中。
